a

Back To Top

This tag is used to indicate a link.

This tag includes the attribute href which allows the browser to link the webpage your code indicates.

<a> href="https://www.yourwebsiteurl.com">The clickable link text here. </a>

img

Back To Topp

This tag is used to indicate an image.

This tag includes the attribute src which stands for source. This links the browser to wherever your image is being stored and allows the computer to retrive and display it.

It also includes the attribute alt which allows you to write text describing the image. This is important incase the browser is unable to laod or image, or in the case someone is using a screen reader--the description can be read to them.

This is how it is written: <img src="YourImageSourceLinkHere" alt="A Description of Your Image Here">

html

Back To Top

This tag tells the browser that this is an HTML document, which allows the browser to read and display the code. It is the "container" for the rest of the html code.

This is how it is written: <html>The Rest of Your Code Here </html>

head

Back To Top

Any information contained in the head is information that's important to the code, but isn't displayed on the webpage, such as linking the code to the index and css folders.

This is how it is written: <head>Code Important to the Head Here</head>

link

Back To Top

This tag links an html document and an external resource, such as your index or css folder. This tag goes in the head.

This tag includes the attribute rel which stands for relationship. This defines the relationship between the current document and the linked document.

It also includes the attribute type which defines the sort of document you are linking.

This tag includes the attribute href which allows the html document to link to and read the external resource.

This is how it is written: <link> rel="Relationship of Documents" type="Type of Document You Are Linking" href="Link To Your Folder or Document">

span

Back To Top

The span tag is used for inline styling. It's best used if you just want specific styling applied to one or two words. For example: My cat has green eyes.

This is how it is written: My cat has <span="color:green"> green </span> eyes.

div

Back To Top

On its own, the div tag doesn't mean anything. However, it is used to create "containers" to hold code. It is useful when used in conjuction with CSS for stylizing. It can be used multiple times in a document, but CSS styling will apply to all segments included within the div tag. This is useful if you want lots of the same styling applied throughout.

This is how it is written: <div> Set of Code You Want Grouped for Stylizing. </div>

header

Back To Top

The header is similar to the div. Like the div, it is used as a "container" for stylizating purposes. It typically holds introductory content, logos, authorship information, and/or navigational links. It is often used on the top of the document, however you can use more than one throughout the html code. Like div, any stylizing applied will apply to all segments contained in the header tag.

This is how it is written: <header> Information and Code You Want Grouped in the Header. </header>

nav

Back To Top

Nav is short for navigation. The nav tag is used for navigational links. It is often used to create a menu on a webpage. The anchor tag is included inside the nav to create the links.

This is how it is written: <nav> <a> href="Link to Page"> Clickable Type </a> </nav>

footer

Back To Top

The footer is the opposite of the header. While the header typically holds the information contained at the top of the page, the footer contains the information typically found on the bottom. It often includes copyright, contact, and/or authorship information, as well as links to the top of the page, other parts of the site and/or related documents/webpages.

This is how it is written: <footer> Information and Code You Want Grouped in the Footer. </footer>

main

Back To Top

This tag specifies what the main content of the page is. It should only contain information that is uniqe to that one page, so it shouldn't include things like navigation or side bars, or information like copyright or logos. This element should not be the child of another such as header or div, but it can be the parent of those "containers".

This is how it is written: <main> Set of Code You Want Grouped in the Header. </main>

section

Back To Top

The section tag is similar to the div tag. It is used to create "containers" to hold code. It is useful when used in conjuction with CSS for stylizing. It can be used multiple times in a document, but CSS styling will apply to all segments included within the section tag. This is useful if you want lots of the same styling applied throughout. The section tag can parent other tags such as div, header, or footer.

This is how it is written: <section> Set of Code You Want Grouped. </section>

aside

Back To Top

The aside tag is similar to the div and section tags, but what is typically used for content that is loosely related to the main content. It's comparable to a sidebar in printed documents. It is used to create "containers" to hold code.

This is how it is written: <aside> Related Content </aside>
